Verifying Checksums

Data can sometimes become corrupted during transmission or storage. The larger the file being transmitted or stored, the larger the risk of corruption. How can you tell if a file has been corrupted?

As a way of checking against such corruption, you can compute something called a checksum or hash sum. A checksum is a single number (usually expressed in hexadecimal) that is unique to the file for which it has been computed. By checking this number at a later date, or after the file has been transmitted, you can verify the integrity of the data by comparing it to the original checksum.

Why does this matter for science?

Scientists often deal with large datasets. When I run a computer simulation of star formation, I may produce 10 GB of output for analysis. These output files are stored on a server somewhere in southern Ontario. To analyze these files, I need to have them on my desktop. I may be disinclined to transfer 10 GB of data raw, so I might compress them first, transfer, decompress, and read. When 10 GB are involved, there's room for error somewhere along that pipe.

Also, you may notice that when you download some large pieces of software, like a disk image of the Ubuntu linux distribution, you will be provided with an md5 checksum. You'd hate to download a 2GB disk image, burn it to a DVD, then realize it was corrupted just as you tried to install Ubuntu on your friend's machine. You should have verified the checksum after downloading.

Common types of checksum

The md5 checksum is popular, but it's being discouraged after cryptologists found "collisions" for the hash function used to compute the md5 checksum. A collision is when two separate pieces of information produce the same checksum. Remember how I said that a checksum was supposed to be unique? Collisions, therefore, are opportunities for security exploits or false positives.

Another type that I've seen on the Google Code repository is the "Secure Hash Algorithm" (version 1), or SHA-1. This was developed by the National SecurityAgency (NSA) as a secure cryptographic hash function. Still, there are plans to phase it out in favor of SHA-2. This is probably more information than the average scientist needs to know---unless you're dealing with secure government information, in which case you already know more than I do.

Computing checksums for your files

This part is actually very easy. Your system probably already has the tools to do this, if you're running a UNIX-like operating system.﻿ Depending on your choice of cryptographic hash, enter the appropriate command at the prompt.

 Perform an MD5 checksum on myfile.txt
 Perform an SHA-1 checksum on myfile.txt
 $ md5sum myfile.txt
$ sha1sum myfile.txt

﻿Your operating system will spit something on the screen like:

﻿$ md5sum myfile.txt

d41d8cd98f00b204e9800998ecf8427e  myfile.txt

$ sha1sum myfile.txt

da39a3ee5e6b4b0d3255bfef95601890afd80709  myfile.txt

Notice that the SHA-1 sum is longer. To perform the checksum on a bunch of files in a directory, just type something like "md5sum *" to get the checksum for everything in a directory. Better yet, record all the checksums in a file.

﻿$ md5sum * > checksums.txt

Keep that checksums.txt file! You can use it later to verify the integrity of your data after a transmission or after long-term storage.

Verifying checksums for your files

Remember how I told you to keep that checksums.txt file? Here's where it comes in handy. You can quickly instruct the md5sum program to check all the files listed in checksums.txt. Here's how:

Suppose my checksums.txt file looks like this:

﻿10a93ff74c9d7b7dcfd8c6e157be13c4  file1.txt
96c3bc08d63a2e592f129575ab91c0d7  file2.txt
048f3308ffa6eb02cb5bb4dd4a247a43  file3.txt
399aa5ad17becd764e95817de70bb899  file4.txt

Then I check the files listed like this:

﻿$ md5sum -c checksums.txt

file1.txt: OK
file2.txt: OK
file3.txt: OK
file4.txt: OK

Can you imagine how this would be important if you're downloading 50 files, totalling 10 GB, and you had to make sure everything made it to the other side OK?

Verifying a checksum for a single file you downloaded

Sometimes you just download a single file from the Google Code repository or some other website. They list the checksum and you just want to check this against the file. One quick way to do this is:

﻿$ md5sum file1.txt | grep 10a93ff74c9d7b7dcfd8c6e157be13c4

If the check is successful, it will print a match. If it fails, it won't print anything.
